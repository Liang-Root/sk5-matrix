# ⚙️ PVE LXC 容器开启 TUN 权限指南 (前置必做)

在部署“六脉神剑 V9.0”或任何使用 Mihomo 透明网关（TUN 模式）的架构前，**必须**在 PVE 宿主机上为对应的 CT 容器开放 TUN 设备的访问权限。
如果不做这一步，Mihomo 将无法创建底层的虚拟网卡，导致整个流量矩阵无法工作。

---

## 🛠️ 操作步骤 (在 PVE 宿主机上执行)

### 1. 确认容器 ID 并关机
首先，在 PVE 网页端，找到你准备用来做矩阵的 CT 容器（例如 ID 为 `101` 的 Debian 容器）。
请务必先将该容器 **关机 (Shutdown)**。

### 2. 进入 PVE 宿主机底层终端
点击 PVE 左侧导航栏最顶部的 **数据中心下的物理机节点 (Node)**，然后点击右上角的 **>_ Shell**，进入 PVE 宿主机的底层终端。
*(注意：必须是 PVE 宿主机的终端，**不是** CT 容器的终端！)*

### 3. 修改容器配置文件
使用 `nano` 编辑对应容器的配置文件（请将下面的 `101` 替换为你的实际容器 ID）：

```bash
nano /etc/pve/lxc/101.conf

```

### 4. 注入 TUN 设备透传权限

使用方向键将光标移动到文件的**最末尾**，新起一行，完整粘贴以下两行代码：

```text
lxc.cgroup2.devices.allow: c 10:200 rwm
lxc.mount.entry: /dev/net/tun dev/net/tun none bind,create=file

```

*(💡 原理说明：这两行代码的作用，是打破容器的隔离墙，允许该容器读写宿主机底层的 `/dev/net/tun` 虚拟网卡设备。)*

粘贴完成后，按下 `Ctrl + O`，然后敲 `回车` 保存；最后按 `Ctrl + X` 退出编辑器。

### 5. 开机并验证权限

回到 PVE 网页端，将该容器 **开机 (Start)**。
进入该容器的控制台（Console），在容器内部执行以下命令，测试 TUN 权限是否生效：

```bash
cat /dev/net/tun

```

* ✅ **成功标志**：如果终端返回 `cat: /dev/net/tun: File descriptor in bad state`，别慌，这是**绝对正常**的表现！说明 TUN 设备已经成功透传到位。
* ❌ **失败标志**：如果返回 `No such file or directory` 或 `Permission denied`，说明配置没生效，请回去检查配置文件是否填错了，或者容器有没有彻底关机再重启。

---

🎉 **权限开放完毕！** 现在，您可以毫无阻碍地进入该容器，执行 `All-in-One.sh` 脚本，召唤“六脉神剑”了！
